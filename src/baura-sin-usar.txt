// ==============================
// 🌸 Economía de Pétalos (versión simple, sin rifas ni superlikes)
// ==============================
// Reglas:
//  - Publicar una reseña → +5 pétalos al autor
//  - Recibir una reseña en tu negocio → +5 pétalos al dueño del negocio
//  - Dar like a una reseña → −1 pétalo (se quema). Si saldo < 1, no puedes dar like.
//  - Objetivo: ciclo simple de ganar (crear) y gastar (reaccionar).

// ---- PATCH PRISMA ----
// Añade saldo + ledger de transacciones
model PetaloTx {
  id        Int      @id @default(autoincrement())
  usuarioId Int
  delta     Int      // + gana, - gasta
  motivo    String   // 'resena_autor','resena_negocio','like'
  refTipo   String?  // 'Resena','Post'
  refId     Int?
  creadoEn  DateTime @default(now())
  usuario   Usuario  @relation(fields: [usuarioId], references: [id])

  @@index([usuarioId, creadoEn])
  @@index([refTipo, refId])
}

# API v1 – Endpoints propuestos (cobertura total)
_Base path_: `/api/v1`  · Auth via Bearer JWT  · Paginación: `?page=1&limit=20`  · Filtros comunes: `?q=texto&sort=-creadoEn`

## 0) Salud
- `GET /health` → estado del servicio

## 1) Auth
- `POST /auth/registro` → {nombre, nickname, email, password}
- `POST /auth/login` → {email, password}
- `POST /auth/refresh` → refresh token
- `POST /auth/logout`
- `POST /auth/forgot-password` → {email}
- `POST /auth/reset-password` → {token, newPassword}
- `POST /auth/verify-email` → {token}

## 2) Usuario
- `GET  /me` → perfil propio + saldos (pétalos)
- `PATCH /me` → actualizar {nombre, bio, foto}
- `GET  /me/petalos/balance`
- `GET  /me/petalos/tx` → ledger (paginado)
- `GET  /usuarios/:id` → perfil público
- `GET  /usuarios` → listado/búsqueda
- `PATCH /usuarios/:id` (admin)
- `DELETE /usuarios/:id` (admin)

## 3) Categorías & Subcategorías

  - `PATCH /pedidos/:id` → {estado}

  - `GET  /compras/:id`
  - `GET  /me/compras`

  - `POST /compras/:id/pagos` → {metodoPago, cantidad}
  - `GET  /pagos/:id`

## 14) Logros
- `GET  /logros` → filtros: `?tipo=&objetivoTipo=&objetivoId=`
- `POST /logros` (admin)
- `PATCH /logros/:id` (admin)
- `DELETE /logros/:id` (admin)
- `GET  /me/logros` → progreso
- `POST /logros/:id/grant` (admin) → concede manualmente a un usuario {usuarioId}

## 15) Pétalos (economía simple)
- `GET  /me/petalos/balance`
- `GET  /me/petalos/tx` → historial
- (Automático) **on reseña creada** → `+5 autor`, `+5 dueño negocio`
- (Automático) **on like** → `−1` del que da like (si saldo<1 → 409)
- `POST /admin/petalos/ajustar` (admin) → {usuarioId, delta, motivo}
- `GET  /economia/config` (admin)
- `PATCH /economia/config` (admin) → ajustar reglas si hiciera falta

## 16) Búsqueda y Feed
- `GET /search` → busca en negocios/productos/posts (`?q=&tipo=`)
- `GET /feed` → timeline de posts (novedades, reseñas, promos)

## 17) Admin (seguridad/gestión)
- `GET  /admin/metrics` → conteos, M2 de pétalos, DAU, etc.
- `PATCH /admin/usuarios/:id/ban`
- `GET  /admin/auditoria/petalos` → agregados de `PetaloTx`

---
### Notas de implementación
- **Eventos automáticos**: pagar pétalos al crear reseña y cobrar al dar like se hacen en **transacciones** de los endpoints correspondientes.
- **Errores**: usar 400 (validación), 401/403 (auth), 404 (no existe), 409 (conflicto: saldo < 1 para like), 422 (regla negocio), 500 (server).
- **Idempotencia**: pagos con `Idempotency-Key` en cabecera.
- **Rate limiting**: `/auth/*` y `/posts/:id/like`.
- **Paginación**: `X-Total-Count` en cabecera.
